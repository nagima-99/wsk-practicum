# WSK Practicum - Практические работы

Данный репозиторий предназначен для сбора, хранения и проверки практических работ студентов
по веб-технологии.

Все работы сдаются **через отдельные ветки и Pull Request**.
Работа напрямую в ветке `main` запрещена.

---

Каждый студент работает **только в папке со своим именем**.

---

## Пошаговая инструкция

### 1️⃣ Склонировать репозиторий на компьютер

Откройте репозиторий:
[Репозиторий WSK Practicum](https://github.com/nagima-99/wsk-practicum.git)

В терминале выполните:

```bash
git clone https://github.com/nagima-99/wsk-practicum.git
```

Перейдите в папку проекта:

---

### 2️⃣ Создать свою рабочую ветку
Работа в ветке `main` запрещена.

Команда:
```bash
git checkout -b Ivanov_Landing
```

---

### 3️⃣ Найти или создать свою папку
Перейдите в папку:

---

### 4️⃣ Создать папку с практической работой
Внутри своей папки:
- создайте папку с названием задания  
или  
- переместите туда уже готовый проект

---

### 5️⃣ Сохранить изменения (commit)
После добавления файлов выполните:

```bash
git add .
git commit -m "Практика: Landing page"

```
---

### 6️⃣ Отправить ветку на GitHub
```bash
git push origin Ivanov_Landing
```
---

### 7️⃣ Создать Pull Request
1. Перейдите в репозиторий на GitHub
2. Нажмите **Compare & pull request**
3. В описании укажите:
   - ФИО
   - Название работы
4. Нажмите **Create pull request**

---

## 📌 Правила сдачи работ
- ❌ НЕ выполнять `git init`
- ❌ НЕ работать в ветке `main`
- ❌ НЕ изменять и не удалять чужие папки
- ✔️ Одна работа = одна ветка = один Pull Request
- ✔️ Загружать файлы **только в свою папку**
- ✔️ Названия файлов должны соответствовать заданию

---

## 👩‍🏫 Проверка работ
Все Pull Request проверяются.  
Работа считается принятой **только после одобрения Pull Request**.
